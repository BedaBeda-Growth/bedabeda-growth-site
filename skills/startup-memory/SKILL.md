# startup-memory\n\nProduces a compressed session brief by querying the OpenViking ChromaDB memory layer (port 7080) or falling back to direct file reads from exports/openviking/. Run at the start of every Kai Agent session.\n\n## Steps\n1. Check health: bash ~/scceo-1/openviking/health.sh\n2. Query chromadb/exports for daily overview, reds, do-not-retry, megafix domains\n3. Compile startup brief: active domains, blockers, changes since last session, do-not-retry list\n4. Surface brief before any other work begins
